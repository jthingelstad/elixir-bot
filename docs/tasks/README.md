# docs/tasks/

Active long-form design docs and product-team reports.

This is the working area, not an archive:

- **Design docs** — the *why* behind an in-flight arc (linked from its GitHub tracking issue, per `AGENTS.md` → Work Tracking). When the arc ships, move the doc to `docs/archive/`; when a doc describes a stable, ongoing system, move it to `docs/reference/`.
- **Product-team output** — the Data Analyst writes `data-YYYY-MM-DD.md` briefs, the Quality Manager writes `quality-YYYY-MM-DD.md` reports, and the Product Manager writes design docs here (see `scripts/product-team/`).

Keep this directory current. Anything finished belongs in `docs/archive/`.
